Svelte for React Devs

UI Meetup - 20th Apr 2021

By Phani Rahul Sivalenka @rahulsivalenka

This talk introduces Svelte to React devs.

Where Svelte is strong

  • minimal learning curve - as Svelte components are written with almost usual HTML, CSS and JavaScript.
  • less code - allows us to produce real business logic without additional coding complexities.
  • no virtual DOM - makes the app even faster.
  • lightweight - bundles down to around 4 KB when compared to React (around 40KB).
  • faster build times - improves DX.
  • scoped styling out-of-the-box - means no CSS leaking unless we intend to
  • in-built state management - using stores

Where Svelte falls behind

  • limited ecosystem - as it is a relatively new framework, it is not yet adopted widely. The ecosystem of Svelte is still growing and may not fulfill the current requirements of projects.
  • missing advanced features - advanced features such as offered by React context API and Suspense are not yet available in Svelte

Is Svelte ready for production?

For basic web apps and static sites it would be a perfect fit.

For complex web applications we might have to wait a bit for SvelteKit to come out of beta.

It is possible to use Sapper to build complex web applications. However, Sapper is discontinued with their efforts geared towards SvelteKit.
